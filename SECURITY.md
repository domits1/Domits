# Security Policy

## Supported Versions

Domits does not currently publish versioned releases.

Security updates are applied to the actively maintained branches only.
At this time, the following branches are supported:

| Branch      | Supported |
|-------------|-----------|
| acceptance  | ✅ Yes    |
| main        | ✅ Yes    |
| others      | ❌ No     |

---

## Reporting a Vulnerability

If you discover a security vulnerability, **please do not report it via a public GitHub Issue**.

Instead, report security issues responsibly by contacting the Domits team directly [here](https://www.domits.com/contact/).

When reporting a vulnerability, please include:
- A clear description of the issue
- Steps to reproduce the vulnerability (if possible)
- Any relevant logs, screenshots, or proof of concept

### What to expect
- You will receive an acknowledgment as soon as possible
- The issue will be reviewed and investigated by the team
- If accepted, a fix will be prioritized and deployed responsibly
- You may be contacted for additional information if needed

We appreciate responsible disclosure and efforts to help keep Domits secure.

---

## Reporting Non-Security Bugs

If you want to report a **non-security bug** (such as functional issues, UI problems, or unexpected behavior),
please report it using the GitHub **Bug Report** issue template.

Use the following template when creating a non-security bug report [here](https://github.com/domits1/Domits/blob/acceptance/.github/ISSUE_TEMPLATE/bug_report.md).
